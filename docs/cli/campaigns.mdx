<Info>
**Campaign endpoints are legacy.** The recommended workflow is to create draft accounts and posts individually, tag them, then provision and schedule with command endpoints. See [Launch a Campaign](/guides/launch-a-campaign) for the recommended approach.
</Info>

## Create a campaign

```bash
vidjutsu campaign create --config campaign.json
```

The config file specifies the campaign parameters:

```json
{
  "platform": "instagram",
  "accountCount": 3,
  "postsPerAccount": 2,
  "name": "NYC Summer",
  "niche": "street fashion"
}
```

Returns the campaign ID, cost breakdown, and placeholder account/post IDs. Campaign is created in `draft` status with credits reserved.

## List campaigns

```bash
vidjutsu campaign list
```

## Get campaign details

```bash
vidjutsu campaign get cmp_abc123
```

Returns the campaign with all accounts and posts progress.

## Execute a campaign

Start provisioning accounts and warming:

```bash
vidjutsu campaign execute cmp_abc123
```

Transitions from `draft` to `executing`.

## Cancel a campaign

Cancel and refund reserved credits:

```bash
vidjutsu campaign cancel cmp_abc123
```

Only works for campaigns in `draft` or `executing` status.
