The VidJutsu CLI runs the clone workflow and the calls around it from your terminal or an agent workspace. It is a lightweight wrapper around the [REST API](/api-reference/introduction) — every command maps to a call to `api.vidjutsu.ai`.

## Install

```bash
curl -fsSL https://vidjutsu.ai/install.sh | bash
```

See [installation](/cli/install) for details and manual install options.

## Clone a video

Create a reusable character once, then clone any video onto it.

```bash
# Create a character once. This prints a char_... id.
vidjutsu clone character --prompt "A friendly host in their late 20s, casual streetwear"

# Clone a TikTok video onto that character.
vidjutsu clone run "https://www.tiktok.com/@creator/video/123456789" \
  --character char_2a91b7f4
```

See [`vidjutsu clone`](/cli/clone) for every subcommand and flag.

| [`vidjutsu check`](/cli/check) | Validate a spec against rules |
| [`vidjutsu overlay`](/cli/overlay) | Burn a text overlay onto a video |
| [`vidjutsu upload`](/cli/upload) | Upload media to the CDN |
| [`vidjutsu account`](/cli/accounts) | Manage account records |
| [`vidjutsu post`](/cli/posts) | Manage post records |
| [`vidjutsu reference`](/cli/reference) | Manage reference records |
| [`vidjutsu asset`](/cli/asset) | Manage asset records |
| [`vidjutsu jobs`](/cli/status) | Inspect a durable job |
| [`vidjutsu usage`](/cli/usage) | Remaining daily capacity per endpoint |
| [`vidjutsu subscribe`](/cli/subscribe) | Manage your subscription |
| [`vidjutsu info`](/cli/info) | Show API discovery info |

## Configuration

The CLI stores its config at `~/.vidjutsu/config.json`:

```json
{
  "apiUrl": "https://api.vidjutsu.ai",
  "apiKey": "vj_live_xxxx"
}
```

All output is JSON by default, which makes it easy to pipe into `jq` or other tools.
