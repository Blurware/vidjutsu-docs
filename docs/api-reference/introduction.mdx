## Base URL

```
https://api.vidjutsu.ai
```

## Authentication

Most endpoints require an API key passed in the `X-Api-Key` header:

```bash
curl https://api.vidjutsu.ai/v1/accounts \
  -H "X-Api-Key: vj_live_your_key_here"
```

Get your API key by [purchasing credits](/authentication) with any card via Stripe.

## Tags

Accounts and posts support AWS-style key-value tags for organization and filtering:

```json
{
  "tags": [
    { "key": "brand", "value": "summer-launch" },
    { "key": "region", "value": "northeast" }
  ]
}
```

Filter on GET requests: `?tag.brand=summer-launch`

## Response codes

| Code | Description |
|------|-------------|
| `200` | Success (read operations, edits) |
| `201` | Created (new records) |
| `202` | Accepted (async operation started) |
| `400` | Missing or invalid parameters |
| `401` | Missing or invalid API key |
| `402` | Insufficient credits |

## Error format

All errors return a consistent JSON structure:

```json
{
  "error": "insufficient_credits",
  "message": "This operation requires 10 credits but you have 2",
  "setup": "/v1/api_keys"
}
```

The `setup` field is included on `402` errors to help agents self-recover by initiating a top-up.

## Synchronous responses

All intelligence endpoints (`/v1/watch`, `/v1/extract`, `/v1/transcribe`, `/v1/check`) return synchronous `200` responses with the result in the body. No polling required.

## CORS

The API allows cross-origin requests from any origin. Allowed headers: `Content-Type`, `X-Api-Key`, `Authorization`.

## Versioning

Responses include the `VidJutsu-Version` header (`2026-03-25`) indicating the current API version.

## Rate limits

No rate limits are enforced during beta. Fair use applies.
