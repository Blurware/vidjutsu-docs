The VidJutsu SDK is a typed TypeScript client generated from the [OpenAPI spec](https://raw.githubusercontent.com/tfcbot/vidjutsu-openapi/main/openapi/spec.json). It covers the clone workflow (check, characters, starting image, video) plus the steps around it: discovery, download, watch, transcribe, compliance, overlay, and job polling.

## Install

```bash
npm install vidjutsu
```

## Quick start

Create a reusable character once, then clone a video onto it.

```typescript
import { createClient } from "vidjutsu";

const vj = createClient();

// 1. Create a reusable character (returns { id: "char_...", imageUrl, model }).
const { data: character } = await vj.createCharacter({
  prompt: "A friendly host in their late 20s, casual streetwear",
});

// 2. Stage the source video, then score it.
const { data: source } = await vj.downloadTikTokVideo({
  url: "https://www.tiktok.com/@creator/video/123456789",
});
const { data: check } = await vj.cloneCheck({ videoUrl: source.url });
console.log(check.verdict, check.score);

// 3. Compose a starting frame, generate the clone, and poll for the result.
const { data: starting } = await vj.cloneStartingImage({
  characterId: character.id,
  sourceVideoUrl: source.url,
  prompt: "Match the framing and pose of the source's opening frame",
});
const { data: task } = await vj.cloneVideo({
  startingImageUrl: starting.imageUrl,
  sourceVideoUrl: source.url,
  model: "kling",
});
const { data: result } = await vj.getCloneVideo(task.id);
console.log(result.videoUrl);
```

## Clone methods

| Method | Description |
| --- | --- |
| [`cloneCheck`](/sdk/clone-check) | Score whether a source video will clone well. |
| [`createCharacter` / `listCharacters` / `getCharacter`](/sdk/characters) | Create and reuse persisted characters. |
| [`cloneStartingImage`](/sdk/clone-starting-image) | Compose a clean character-swapped starting frame. |
| [`cloneVideo` / `getCloneVideo`](/sdk/clone-video) | Generate a clone video and poll the task. |
| [`downloadTikTokVideo` / `downloadInstagramVideo`](/sdk/download) | Stage a single source video to the CDN. |

## Supporting methods

| Method | Description |
| --- | --- |
| [`scrapeTikTokProfile` / `scrapeInstagramProfile` and related `scrape*` methods](/sdk/scrape) | Pull structured profile, post, comment, and search data. |
| [`watchMedia`](/sdk/watch) | Run prompted video or image understanding. |
| [`transcribeMedia`](/sdk/transcribe) | Speech-to-text with word-level timing. |
| [`checkComplianceVideo` / `checkCompliancePrompt`](/sdk/check) | Run an ad-compliance check. |
| [`createOverlay`](/sdk/overlay) / [`createDisclaimer`](/sdk/disclaimer) | Burn a text overlay or disclaimer onto a video. |
| `getJob` / `getDistributionJob` | Poll a durable job to completion. |

## Authentication

The client resolves your API key in order:

1. **Explicit** — `createClient({ apiKey: "vj_live_..." })`
2. **Environment variable** — `VIDJUTSU_API_KEY`
3. **Config file** — `~/.vidjutsu/config.json` (shared with the [CLI](/cli/auth))

```typescript
const vj = createClient();
```

Use a tenant-scoped key in a trusted environment. Do not place a key in a prompt, skill file, or public client bundle.
