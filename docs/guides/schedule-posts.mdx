Once you have an [active account](/guides/manage-accounts) and your video ready, create post drafts and schedule them when you're ready.

## Draft-first workflow

Post creation is now a two-step process:

1. **Create a draft** (free) — Set up the post content without spending credits.
2. **Schedule** (36 credits) — Charge credits and queue the post for publishing.

## Step 1: Create a draft post

<CodeGroup>
```bash CLI
vidjutsu post create --video vid_q9f3 --caption "Summer fits"
```

```bash API
curl -X POST https://api.vidjutsu.ai/v1/posts \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: vj_live_abc123..." \
  -d '{
    "accountId": "acc_8k2x",
    "videoId": "vid_q9f3",
    "caption": "Summer fits",
    "scheduledAt": "1711641600",
    "tags": [
      { "key": "campaign", "value": "summer-launch" },
      { "key": "content-type", "value": "ugc" }
    ]
  }'
```
</CodeGroup>

**Cost:** 0 credits (free). The `accountId` is optional at draft time.

```json
{
  "id": "post_m2v1",
  "status": "draft",
  "creditsCharged": 0
}
```

## Step 2: Edit the draft (optional)

Update a draft post's content before scheduling:

```bash
curl -X PUT "https://api.vidjutsu.ai/v1/posts?id=post_m2v1" \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: vj_live_abc123..." \
  -d '{
    "caption": "Updated caption",
    "videoUrl": "https://new-video-url.com/video.mp4",
    "tags": [
      { "key": "campaign", "value": "summer-launch" },
      { "key": "content-type", "value": "ugc" },
      { "key": "batch", "value": "week-1" }
    ]
  }'
```

<Warning>
PUT only works on posts in `"draft"` status. Once scheduled, the post content is locked.
</Warning>

## Step 3: Schedule the post

When you're ready, schedule the post for publishing:

```bash
curl -X POST https://api.vidjutsu.ai/v1/posts/schedule \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: vj_live_abc123..." \
  -d '{ "postId": "post_m2v1" }'
```

**Cost:** 36 credits ($3.60).

**Requirements:**
- Post must have an `accountId` set
- Account must be in `"active"` status

```json
{
  "id": "post_m2v1",
  "status": "scheduled",
  "creditsCharged": 36
}
```

If `scheduledAt` was set on the draft, the post will publish at that time. If not, it publishes as soon as possible.

## Tags

Posts support AWS-style key-value tags for organization:

```json
{
  "tags": [
    { "key": "campaign", "value": "summer-launch" },
    { "key": "content-type", "value": "ugc" }
  ]
}
```

Tags are set at creation and updatable via PUT on drafts. Filter posts by tag on the list endpoint:

```bash
curl "https://api.vidjutsu.ai/v1/posts?tag.campaign=summer-launch" \
  -H "X-Api-Key: vj_live_abc123..."
```

## Post statuses

| Status | Description |
|--------|-------------|
| `draft` | Created for free, editable via PUT |
| `scheduled` | Credits charged, queued for publishing |
| `publishing` | Currently being posted |
| `published` | Successfully posted |
| `failed` | Post failed (check account status) |
| `cancelled` | Post was cancelled |

## Cancel a post

<CodeGroup>
```bash CLI
vidjutsu post cancel post_m2v1
```

```bash API
curl -X DELETE https://api.vidjutsu.ai/v1/posts/post_m2v1 \
  -H "X-Api-Key: vj_live_abc123..."
```
</CodeGroup>

Only posts with `"draft"` or `"scheduled"` status can be cancelled.
