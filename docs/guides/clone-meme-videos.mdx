VidJutsu is designed for short, one-performer meme references that can be recreated with a brand character while preserving the source&apos;s timing, sound, and text-video format.

## Use an agent task

For the complete workflow, submit one task and tail the events:

```typescript
const task = await client.agentTasks.create(
  {
    prompt: "Clone this meme video for our brand with our character. Keep source timing and sound. Replace the wall text with: When our product finally clicks.",
    inputs: [{ name: "source_video", url: sourceVideoUrl }],
  },
  { idempotencyKey: "brand-meme-001" },
);
```

Use a stable idempotency key for the business request. If a worker retries, VidJutsu resumes checkpoints instead of repeating completed paid stages.

## Workflow stages

1. **Import** — TikTok or Instagram references are downloaded once to a stable VidJutsu asset.
2. **Clone check** — source evidence is inspected before work begins. A poor fit returns reasons and stops.
3. **Character** — a reusable brand character is created or reused.
4. **Starting image** — the first frame is rebuilt with the brand character and no source wall text.
5. **Motion clone** — the requested Seedance or Kling motion-control model follows the reference performance.
6. **Overlay and QA** — replacement wall text is applied after motion generation and the final video is verified.

## Good references

- Under 15 seconds
- One visible primary subject
- A continuous or simple shot
- Clear subject visibility and limited occlusion
- Simple gestures and limited props or interactions
- Readable, replaceable wall text

## Model selection

`/v1/clones/video` requires an explicit model:

- `kling-motion-control` for references where closely following source movement matters.
- `seedance` when the creative request calls for that provider.

VidJutsu does not switch model families silently. If clone check is below the passing threshold, the workflow stops unless an authenticated override includes a reason.

## Artifact lineage

Every completed workflow returns the final asset and the intermediary lineage through task events and job records. Use that lineage to display progress, retry only an unfinished stage, or let a human review an artifact.
